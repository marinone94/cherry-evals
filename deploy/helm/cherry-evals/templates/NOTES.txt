API Endpoint
------------
The API is running inside the cluster. To access it locally:

  kubectl port-forward -n {{ .Release.Namespace }} svc/{{ include "cherry-evals.fullname" . }}-api {{ .Values.api.port }}:{{ .Values.api.port }}

Then open: http://localhost:{{ .Values.api.port }}
Docs:      http://localhost:{{ .Values.api.port }}/docs
Health:    http://localhost:{{ .Values.api.port }}/health

Configuration
-------------
Set API keys via Helm values or a values override file. Example:

  helm upgrade cherry-evals deploy/helm/cherry-evals \
    --set config.googleApiKey=<key> \
    --set config.anthropicApiKey=<key> \
    --set config.cerebrasApiKey=<key> \
    --set postgres.password=<strong-password>

For full documentation see: https://github.com/marinone94/cherry-evals
